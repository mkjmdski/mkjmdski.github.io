HCL && "${interpolation}"

  • HCL is compromise between human-friendly and machine-friendly configurations
  • HCL is fusion of full programing languages and data structure languages
  • HCL is JSON-compatible - you can generate JSON and run it as HCL
  • HCL supports comments and <<EOF syntax for multiline strings.

Exact specification on the language can be found in the Hashicorps repository

---

Logical statements

  • if: subnet = "${var.env == "production" ? var.prod_subnet : var.dev_subnet}"
  • comprassions and boolean logic: == != > < >= <=
  • looping (0.11): ${count.index} -> creating ++like loops
  • looping (0.12): %{ for instance in aws_instance.example ~}server ${instance.id}%{ endfor }. -> templating
---

References

  • vars: "${var.aws_account_number}" (0.11), var.aws_account_number (0.12)
  • maps: "${var.aws_account_number['client1']}"
  • lists: "${var.aws_account_number[count.index]}"
  • single resources/data: "${data.aws_ec2_instance.single.ipv4}"
  • multiple: "${data.aws_ec2_instance.cluster.*.ipv4}"
  • modules: "${module.module_name.output_name.key}"
---

Functions

  • Math functions: floor, abs, log, pow, max, signum, ${2 + 2}
  • String functions: base64, concat, chomp, format, indent, join, lower
  • Data types operations: chunklist, element, map, list, jsonencode, keys, sort, slice
  • System calls: dirname, file, timestamp

All interpolations can be found in the docs.
